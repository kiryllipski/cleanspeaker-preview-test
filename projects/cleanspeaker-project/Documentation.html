Everything an Android engineer needs to ship the prototype as a production app: audio engine, haptic feedback map, cleaning algorithms, permissions, and premium gating.

Platform Android 9+ (API 28) Target Android 14 (API 34) Stack Kotlin + Compose Audio Oboe / AAudio

Overview

What we’re building

CleanSpeaker is a single‑purpose Android utility that uses precisely tuned sound and haptic patterns to physically eject water, loosen dust, and unclog phone speakers. It monetises on the first session: a 5‑step diagnostic onboarding builds trust, surfaces a quantified rescue plan, and routes to a special‑offer paywall with a downsell fallback.

Source of truth

The 23 screens in All Screens.html are the visual spec. Pixel values, gradients, and motion are all derived from tokens.css and component code in primitives.jsx — do not re‑invent. Component dimensions and animation timings in the prototype are intentional.

Goals (in priority order)

  • First‑session conversion. Trial‑start rate from cold install. Track the funnel end‑to‑end.
  • Perceived efficacy. The cleaning cycle must feel like it’s doing something — audio, haptics, and animation are co‑designed.
  • Trustworthy diagnostic. No fake numbers. The score is derived from a real sound‑test result, even if simple.
  • Premium tool aesthetic. Dark, calm, electric‑cyan. Not a cheap cleaner app.

Audio engine

Output path, channels, levels

All cleaning cycles play through the loudspeaker — never the earpiece. Force routing on cycle start, and refuse to start if headphones or BT audio are connected.

// AudioManager force speaker output (API 31+)
val audio = getSystemService(AudioManager::class.java)
val speaker = audio.availableCommunicationDevices
    .first { it.type == AudioDeviceInfo.TYPE_BUILTIN_SPEAKER }
audio.setCommunicationDevice(speaker)
audio.mode = AudioManager.MODE_NORMAL

// Refuse to start if rerouted
if (audio.isBluetoothA2dpOn || audio.isWiredHeadsetOn) {
    showScreen(AudioWarning)
    return
}

Output spec

  • Sample rate — 48 kHz (native on most Android speakers; resample once if device prefers 44.1 kHz).
  • Bit depth — 16‑bit PCM, mono.
  • Buffer — 480 frames (≈ 10 ms) for Oboe low‑latency callback.
  • Level — Generate at −3 dBFS peak. The system volume is what the user controls — never auto‑set it.
  • Anti‑pop — 50 ms linear ramp on stage start, 80 ms ramp on stage end. Without ramps, abrupt sine onset produces audible click that breaks the premium feel.

Hearing safety

Cap the generator at −3 dBFS and refuse to start if system media volume is > 90%. Show the AudioWarning screen with copy: “Lower the volume before cleaning — this is loud.”

Frequency sweep math

Sine generation & phase continuity

Each cleaning stage is a continuous sine sweep around a target frequency with a slow ±8 % modulation envelope to keep the speaker membrane moving through varied resonance points (real-world water and dust ejection in industry literature uses similar approaches).

// Continuous-phase sine. Phase MUST persist across buffers,
// otherwise you get audible clicks every 10 ms.
class SineGen(val sr: Int = 48000) {
    private var phase = 0.0
    private var t = 0.0

    fun fillBuffer(out: ShortArray, hz: Double, ampDb: Double) {
        val amp = Math.pow(10.0, ampDb / 20.0) * Short.MAX_VALUE
        for (i in out.indices) {
            // ±8% modulation — sweeps through speaker resonance
            val mod = 1.0 + 0.08 * Math.sin(2.0 * Math.PI * t * 0.7)
            val fInst = hz * mod
            phase += 2.0 * Math.PI * fInst / sr
            if (phase > 2.0 * Math.PI) phase -= 2.0 * Math.PI
            out[i] = (Math.sin(phase) * amp).toInt().toShort()
            t += 1.0 / sr
        }
    }
}

Cleaning algorithms

Cycle state machine

The Deep Clean cycle is a deterministic 3‑stage state machine. Use a coroutine‑backed Flow<CycleState> so the UI (dial, waveform, progress) and the audio thread observe the same source of truth.

sealed class CycleState {
    data class Stage(val n: Int, val hz: Double,
                     val remaining: Duration) : CycleState()
    object Paused : CycleState()
    data class Done(val healthDelta: Int) : CycleState()
    object Aborted : CycleState()
}

fun runDeepClean(): Flow<CycleState> = flow {
    val stages = listOf(
        StageDef(165.0, 12.seconds),
        StageDef(300.0, 18.seconds),
        StageDef(500.0, 10.seconds),
    )
    for ((i, s) in stages.withIndex()) {
        audio.setTargetHz(s.hz)
        val start = clock.now()
        while (clock.now() - start < s.duration) {
            emit(CycleState.Stage(i+1, s.hz, s.duration - (clock.now()-start)))
            delay(50) // 20 fps UI tick
        }
    }
    emit(CycleState.Done(healthDelta = computeDelta()))
}

Abort handling

The big “Stop” button mid‑cycle must show ScrAbortModal (screen 22), not abort silently. Single‑tap pause is fine; full stop needs confirmation because users will accidentally tap during the Hz dial interaction.

Result scoring

The score in ScrResult is derived from the post‑cycle Sound Test — not a fixed number. Compute as a weighted average of three measurements:

  • Loudness — peak SPL recorded via the mic during a 20–20 000 Hz sweep (40 %).
  • Clarity — RMS energy in 1–4 kHz band / total RMS (40 %).
  • Balance — abs(L − R) dB, inverted and clipped (20 %).

Pulse Clean (haptic mode)

Vibration as cleaning vector

Pulse Clean (screen 18) is the haptic counterpart to sound cleaning. It uses the device’s linear‑resonant‑actuator (LRA) to physically shake the chassis and dislodge debris stuck under the speaker grille. It can layer with a 165 Hz tone for combined sound + mechanical attack.

Safety

  • Hard‑cap pattern duration at 30 s per session (battery + chassis heat).
  • Disable while device temperature is > 38 °C (read via PowerManager.getCurrentThermalStatus).
  • Pause vibration during phone call / alarm (listen to AudioFocus).
  • Pulse Clean is a Pro feature — show ScrLocked on tap for free users.

Universal rules

  • Always check vibrator.areAllPrimitivesSupported(...) first; fall back to EFFECT_TICK / EFFECT_CLICK if unsupported.
  • Respect system “Touch feedback” setting — read Settings.System.HAPTIC_FEEDBACK_ENABLED before firing.
  • Suppress all non‑essential haptics when battery saver is on.
  • Honour the “Haptic feedback” toggle in ScrSettings (screen 20).

Premium gating

Free vs Pro

Free

  • 1 cleaning cycle per 24 hours
  • Water Eject only (no Dust, Muffled, Pulse)
  • Basic Sound Test (no frequency curve detail)
  • Default 3‑stage cycle (no Manual Hz)
  • Limited result history (last 1)
  • Banner ad before Result screen

Pro

  • Unlimited cleaning cycles
  • All 4 quick actions + Pulse Clean haptic mode
  • Full Sound Test with L / R balance & frequency curve
  • Manual Hz tool (20 Hz – 20 kHz)
  • Unlimited history + auto‑clean reminders
  • No ads

Gating implementation

enum class Entitlement { FREE, PRO_TRIAL, PRO }

fun canAccess(feature: Feature): Boolean = when (feature) {
    Feature.WATER_EJECT -> true
    Feature.DUST_CLEAN,
    Feature.MUFFLED_FIX,
    Feature.DEEP_CLEAN,
    Feature.PULSE_CLEAN,
    Feature.MANUAL_HZ        -> entitlement != Entitlement.FREE
    Feature.UNLIMITED_CYCLES -> entitlement != Entitlement.FREE
                            || cyclesToday() < 1
}

Paywall logic

  • Main paywall (screen 09) shows after onboarding flow finishes. 3 plans, Annual pre‑selected, 7‑day trial. Countdown timer is real — persist start time in DataStore, recompute on each launch.
  • Close‑intent → downsell paywall (screen 10). Triggered by back press / X button. Shows a single $4.99/wk plan vs $9.99 baseline, 5‑min countdown. Fire only once per session — if user dismisses both, gate the locked features for the rest of the session.
  • Mid‑app upsell — tapping a Pro feature as free user opens ScrLocked (screen 21). “Start free trial” routes to main paywall, “Maybe later” closes the modal.
  • Restore purchases available from Settings (screen 20). Implement via BillingClient.queryPurchasesAsync.

Subscription compliance

  • Both paywalls must show price, billing period, and renewal terms before the CTA — currently rendered as “Auto‑renews · cancel anytime” small text under the button.
  • Trial price & conversion price must be equally legible (Google Play 2024 policy).
  • “Restore purchases” must be accessible without account creation.

State persistence

  • Onboarding answers — kept in SavedStateHandle only; not persisted past install. If user kills the app mid‑onboarding, restart.
  • Cycle history — DataStore, capped at last 50 entries. Each entry: timestamp, action, duration, score delta.
  • Entitlement — DataStore + every‑launch Play Billing verification. Treat cached state as soft; do not gate based on it alone for > 24 h.
  • Settings toggles — DataStore. Read on every relevant action (haptic enable, notifications).

QA checklist

Pre-release

  • Audio routing. Plug AirPods mid‑cycle — playback aborts, AudioWarning shown.
  • Phase continuity. Listen to a full cycle on headphones at low volume; should be click‑free at every stage boundary.
  • Volume safety. Set system media to 100% — app gates start with a warning.
  • Haptic fallback. Test on a device without composition primitives (e.g. Android 11 emulator); all haptics fall back gracefully.
  • Pulse heat. Run Pulse Clean back‑to‑back for 60 s — app self‑pauses on thermal alert.
  • Trial countdown. Background app for 6 h, reopen — timer is correct, not reset.
  • Restore. Uninstall, reinstall, restore — Pro features unlock without account.
  • Permission deny. Decline audio at install — app stays in ScrPermission until granted.
  • Offline. Toggle airplane mode on home — banner shows, cleaning still works (no network needed), paywall gracefully degrades.
  • Battery saver. Enabled → non‑essential animations & haptics suppressed.
  • TalkBack. All buttons have content descriptions, dial values are spoken.
  • Tablet / fold. Layouts gracefully clip at > 600 dp — not stretched. Single‑pane only.
  • Dark mode only. No light theme exists by design — confirm no light‑mode token leakage.
