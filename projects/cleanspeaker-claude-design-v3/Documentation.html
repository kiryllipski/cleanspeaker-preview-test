Documentation

Overview

CleanSpeaker is a premium consumer utility for Android that uses precisely tuned audio tones — and a haptic pulse mode — to dislodge water, dust and debris from phone speakers. Business objective is first-session monetization: the onboarding earns trust through diagnostic feedback, then converts to a 3-day trial.

Target user

Android owner who's just dropped, splashed or noticed muffled audio on their phone. Wants relief in under 60 seconds.

Conversion KPI

≥ 8% trial-start rate on the first session. ≥ 35% trial → paid conversion.

Premium gates

Pulse Clean, Manual Hz, unlimited cycles, no ads. Free tier gets one Deep Clean per week.

Platforms

Android 9+ (API 28). Kotlin + Jetpack Compose. Google Play Billing v6.

Tech stack

Native Android, Compose-first. Audio is the critical path — keep the latency budget under 50ms.

User flow

23 screens. First-session path: 11 screens, ~90 seconds.

First session

  1. Permission — explicit audio + vibration consent (no system dialog yet, education first).
  2. Welcome — single CTA, no copy load.
  3. Q1 / Q2 / Q3 — problem, timing, device. Tap-only, no typing required.
  4. Analyzing — 5s scripted diagnostic with progressive checklist. Builds perceived value.
  5. Rescue plan — personalized severity score + 3-stage prescription.
  6. Rating — soft ask (skippable). 5★ taps trigger in-app review API.
  7. Paywall — 3 plans, default to annual. Close → downsell.
  8. Downsell — half-price week with single CTA.
  9. Push prompt — only after paywall to protect notification opt-in.

Returning session

Home → Quick action or Deep Clean → Cycle → Result → Sound test (optional). Premium gates redirect to Locked modal which routes back to Paywall.

Audio engine

Real-time pure-tone synthesis at the OS audio sample rate. The engine streams a sine wave to the device speaker with frequency modulation envelopes per stage.

Tone generator

class ToneGenerator(val sampleRate: Int = 48000) {
  private var phase = 0.0
  private var freq = 165.0

  fun setFrequency(hz: Double) {
    freq = hz.coerceIn(20.0, 20_000.0)
  }

  fun renderBlock(buffer: FloatArray) {
    val twoPi = 2 * Math.PI
    // per-sample frequency interpolation for clean sweeps
    for (i in buffer.indices) {
      buffer[i] = sin(phase).toFloat() * 0.85f
      phase += twoPi * freq / sampleRate
      if (phase > twoPi) phase -= twoPi
    }
  }
}

Output configuration

  • Stream: USAGE_MEDIA with CONTENT_TYPE_SONIFICATION
  • Buffer: 2× getOptimalBufferSize() for under-runs safety
  • Routing: force speaker via setCommunicationDevice() on API 31+
  • Volume: ramp from current to max over 1.5s when Auto-volume is on; restore on exit
Hearing safety. Cap output at −6 dBFS. Skip frequencies in the 2 kHz – 5 kHz peak-sensitivity band during ambient cleaning. Show a peripheral-audio warning if headphones are detected.

Pulse Clean implementation

The Pulse Clean mode drives VibrationEffect as the primary cleaning actuator — sound is layered on top when combine is on. Patterns are composed of primitives.

// 4 patterns, each a VibrationEffect.Composition
val steady = composition {
  addPrimitive(PRIMITIVE_LOW_TICK, intensity, 0)
  // repeat continuously at 40ms intervals
}
val wave = composition {
  // modulated intensity over 800ms loop
  addPrimitive(PRIMITIVE_LOW_TICK, 0.3f, 0)
  addPrimitive(PRIMITIVE_LOW_TICK, 0.7f, 100)
  addPrimitive(PRIMITIVE_LOW_TICK, 1.0f, 200)
  addPrimitive(PRIMITIVE_LOW_TICK, 0.7f, 300)
  addPrimitive(PRIMITIVE_LOW_TICK, 0.3f, 400)
}
val burst = composition {
  addPrimitive(PRIMITIVE_THUD, intensity, 0)
  addPrimitive(PRIMITIVE_THUD, intensity, 1200)
}
val knock = composition {
  addPrimitive(PRIMITIVE_THUD, 1.0f, 0)
  addPrimitive(PRIMITIVE_THUD, 1.0f, 500)
}
Devices without primitives (API < 30 or older OEMs) fall back to amplitude patterns via VibrationEffect.createWaveform() with timings tuned per pattern.

QA checklist

  • Cycle audio plays through bottom speaker even when media volume is at 0 (force volume ramp)
  • Headphones / BT speaker detected → show audio warning before cycle
  • Interruption from incoming call pauses the cycle, resumes on call end
  • Foreground service notification appears within 200ms of cycle start
  • Screen stays on during cycle (FLAG_KEEP_SCREEN_ON); released on exit
  • Haptics respect system Do Not Disturb
  • Paywall close → Downsell appears once per install
  • Trial countdown survives device timezone change & cold start
  • Restore purchases succeeds for users signed in with a different Google account
  • Offline state preserves last-known health score
  • Dynamic font scaling (up to 130%) does not overflow any CTA
  • TalkBack reads every Hz value as "one-six-five hertz", not "one-hundred-sixty-five hertz"
